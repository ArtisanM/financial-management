# 阿里云短信接入指引 · 家庭账房 v0.4.14

> 给你的 step-by-step 操作手册 · 跟着做即可拿到 4 个值填进 `/admin/reminders`:
> **AccessKey ID / AccessKey Secret / 签名(SignName)/ 短信模板 CODE(TemplateCode)**。

## 🚨 阅读须知 · 别踩这两个坑

**(1) 你会在 3 个不同的控制台之间来回切**(我之前没说清,你在「短信服务开发者文档」里找不到 AccessKey 申请就是因为它归 RAM 管):

| 产品 | 在哪 | 做什么 |
|---|---|---|
| 阿里云账号 | account.aliyun.com | 注册 / 实名 / 充值 |
| 短信服务 SMS | dysms.console.aliyun.com | 开通 / 申请签名 / 申请模板 |
| 访问控制 **RAM** | ram.console.aliyun.com | **创建子账号 / 创建 AccessKey / 授权** |

**(2) 个人实名账号 · 签名报备不通**(2026 阿里云官方政策 · 原文:"在当前的短信签名实名制要求下,**个人账号的自用资质无法通过签名实名制报备**"· 见 [getting-started 页](https://help.aliyun.com/zh/sms/getting-started/get-started-with-sms))。
所以:**必须企业实名认证账号**。本指引下文按企业账号写。

**(3) 签名审批通过 ≠ 立刻能发** —— 阿里云审核通过后,签名还要**运营商报备**(三大运营商各自独立报备):
- **官方公布耗时**:5-7 工作日 · 部分 7-10 工作日 · 运营商不承诺时效
- **三家独立并行**:移动/联通/电信各自审,任意一家通过即可对该家用户发送
- **报备期间使用该签名调 SendSms**:接受但下行失败 · 控制台显示「签名实名制报备问题」· 错误码 `PORT_NOT_REGISTERED`
- **没有官方加急通道** · 提升通过率/速度只能靠:① 签名来源选「企事业单位全称或简称」② 签名命名严格匹配资质 ③ 状态变「已报备待验证」后用三家号码各发几条触发系统自动确认
- **驳回后改名只能删旧建新**(SignName 会丢)· 在管理页 ② 段更新 SignName 即可,后端无侵入

## 全链路接口数

- **应用程序运行时调用**:**1 个 API**(`SendSms` · 我们的应用代码已实现 RPC HMAC-SHA1 签名,你不用管)
- **一次性手工配置**:**8 个步骤** · 全在控制台手工做(每步技术上也都有对应 API,但绝大多数人走控制台更省事)

全程预估:**30 - 60 分钟**(若签名 + 模板报备审核顺利);卡在审核时:签名 1-2 小时 · 模板 30 分钟左右(慢则半天)。

## 0. 你需要准备

- **企业实名认证过**的阿里云账号(个人实名走不通 · 见上文 🚨 须知)
- 充值能力:无最低门槛 · 套餐包最低 **6.9 元 / 200 条**(新用户起步价)
- 营业执照扫描件(签名报备要用)

## 1. 注册 + 实名认证

1. 打开 https://account.aliyun.com/register/register.htm 注册账号(手机号 + 邮箱)
2. 登录后 → 头像 → **实名认证**(必须做):https://account.console.aliyun.com/#/secure/realname
   - 个人:身份证 + 人脸识别(几分钟过)
   - 企业:营业执照 + 法人验证(1 个工作日)
3. (可选)绑定支付宝/银行卡,方便后续充值

## 2. 开通短信服务

1. 控制台搜「短信服务」→ 立即开通:https://dysms.console.aliyun.com/overview
2. 同意服务协议、SLA 等(免费开通,不开通就发不了短信)
3. 进入控制台首页能看到「**国内短信**」「国际/港澳台短信」面板,即开通成功

## 3. 充值(可选 · 用按量付费的话连充值都可以晚于报备)

按量付费默认从账户余额扣 · **无最低充值**,你可以先充 10 元试试:

- 路径:https://expense.console.aliyun.com/finance/recharge → 任意金额(微信/支付宝/银行卡)
- 或买套餐包:https://common-buy.aliyun.com/?commodityCode=dysms_pre — 新用户活动 **6.9 元 / 200 条**,够家庭账本用 1-2 年

每月发送量阶梯单价(按量付费 · 2026-05-20 起生效):

| 月发送量 | 系统短信(通知/验证码) | 推广短信 |
|---|---|---|
| ≤ 10 万 | ¥0.045/条 | ¥0.055/条 |
| 10-30 万 | ¥0.040/条 | ¥0.050/条 |
| 30-50 万 | ¥0.039/条 | ¥0.049/条 |

家庭场景全年 ≈ 100 条 · 一年成本约 ¥4.5,可以忽略。

## 4. 申请短信签名 · 这是最容易卡的一步

签名 = 短信开头的 `【xxx】`。运营商要求实名报备,本步审核约 **1-2 小时,失败可重复提交**。

1. 控制台 → 国内短信 → **签名管理** → 添加签名:https://dysms.console.aliyun.com/domestic/text/sign
2. 填:
   - **签名名称**:建议「家庭账房」(2-12 字 · 不含敏感词 · 不含「短信」「通知」)
   - **签名来源**:
     - 企业用户选「企事业单位的全称或简称」+ 上传营业执照
     - 个人用户**只能选**「自有 App 名称 / 网站(需 ICP 备案截图)/ 公众号或小程序」之一,且名称要和签名完全一致
   - **使用场景**:选「通用-验证码 / 通用-通知」(申请通用最容易过)
   - **场景说明**:贴一段示例:`本签名用于家庭记账网页应用,在每月记账截止前 2 天向家庭成员发送提醒短信,通知本期记账截止时间和家庭已完成进度。`
3. 提交 → 等审核(短信通知你结果)
4. **常见拒绝原因 + 解决**:
   - 「签名与申请人主体不符」→ 个人用户用「家庭账房」会被拒 · 改为你 App / 备案站名
   - 「签名包含敏感词」→ 别带「贷」「金融」「投资」「征信」类
   - 「未上传 ICP 备案截图」→ 个人申请域名签名必须上传(用 https://beian.miit.gov.cn 查截图)
   - 「签名与场景说明不符」→ 场景说明写明确点,如上例

## 5. 申请短信模板 · 第二容易卡的一步

模板 = 短信正文格式 · 含变量占位 · 也要审核(约 30 分钟)。

1. 控制台 → 国内短信 → **模板管理** → 添加模板:https://dysms.console.aliyun.com/domestic/text/template
2. 填:
   - **模板名称**:`家庭账本填报提醒`
   - **模板类型**:选「通用-通知」
   - **模板内容**(**严格按本字串复制粘贴**,4 变量必须叫这 4 个名字,顺序也别变):
     ```
     ${brand}账本 ${period} 提醒:距记账截止${days}天,家庭已完成 ${progress},请尽快填报。
     ```
   - **申请说明**:
     ```
     家庭记账网页应用,每月固定窗口向家庭成员发送记账截止提醒。
     变量说明:
       brand    = 家庭别名(如「狗窝」「张家」),≤ 8 字符
       period   = 本期标识(如「2026年5月」「第20周」),≤ 10 字符
       days     = 距记账截止剩余天数(0-15 整数)
       progress = 全家填报进度(如「2/4人」),≤ 8 字符
     发送场景:每月记账截止前 2 天每天 10:00 与 20:00 各一次,仅对未填成员触达。
     ```
3. 提交 → 等审核(短信通知你结果)
4. **常见拒绝原因**:
   - 「变量过多 / 变量内容不明确」→ 阿里云对变量数量没硬限,但每个变量要"说人话",上面的申请说明已写够清楚
   - 「行业场景不明」→ 申请说明写明"家庭账本应用 · 内部成员"
   - 「未声明发送频率」→ 申请说明里写明每天最多 2 次、仅截止前 2 天
   - 「模板内容含敏感词」→ 改个措辞重提

审核通过后,在模板列表能看到 **短信模板 CODE**(形如 `SMS_478235091`)· **抄下来**,§ 7 要用。

## 6. 创建 AccessKey · 安全推荐用 RAM 子账号

**不推荐直接用主账号 AccessKey**(主账号 ak 权限太大,泄露后果严重)· 走 RAM 子账号:

1. 控制台 → 访问控制 RAM → 用户 → **创建用户**:https://ram.console.aliyun.com/users
   - 用户名:`finance-sms`
   - 访问方式:勾「**OpenAPI 调用访问**」(不需要控制台登录)
2. 创建完成后页面会**只显示一次**的 **AccessKey ID** + **AccessKey Secret** · **立刻复制保存到安全的地方**(密码管理器 / 1Password / Bitwarden)· 关掉这个页面就再也看不到 secret 了
3. 给这个子账号授权:点用户名 → 权限 → 新增授权 → 系统策略选 `AliyunDysmsFullAccess`(只给短信权限,不给其它 OSS / ECS 等)
4. (可选)给子账号设 IP 白名单:RAM 用户详情 → 安全设置 → 限制访问 IP · 填你的服务器公网 IP · 进一步降低泄露风险

**主账号 AccessKey 替代方案**(快 · 风险高):头像 → AccessKey 管理 → 创建主账号 AccessKey · 同样会**只显示一次** secret。我**不推荐**这个 —— 但赶时间可以,日后再迁子账号。

## 7. 在 `/admin/reminders` 填入 + 测试

回到家庭账房 → 登录 → **管理 → 提醒**(`/admin/reminders`):

### 7.1 ② 短信渠道

| 字段 | 填什么 |
|---|---|
| 启用短信强提醒 | ✓ 勾上 |
| 短信供应商 | aliyun · 阿里云短信 |
| AccessKey ID | 第 6 步拿到的 AccessKey ID,**直接粘贴** |
| AccessKey Secret | 第 6 步拿到的 AccessKey Secret,**直接粘贴**(保存后再也看不到,但页面会显「已配置(隐藏)」) |
| 短信签名 · SignName | 第 4 步审核通过的签名,**完全一致**(如「家庭账房」)· 不含 `【】` |
| 短信模板 CODE · TemplateCode | 第 5 步审核通过的模板 CODE(亦称「模板编号」),如 `SMS_478235091` |

点「**保存短信配置**」。

### 7.2 ④ 成员手机号

逐个填家里成员的 11 位手机号 · 留空 = 该成员不发短信(仍走站内 banner)。

### 7.3 ③ 一键测试 ⊕

**这是验证全链路是否通的最快方式** —— 选一个成员手机号(或临时输入一个自己的手机号)→ 点「▷ 发送测试短信」。

**成功**:页面顶部绿色 banner 显「✓ 测试短信已发送 · 请查收 · 阿里云 BizId: 905619...」· **几秒到 1 分钟内**该手机号收到测试短信。

**失败**:红色 banner 显阿里云返回的 Code,常见对照:

| Code | 含义 | 怎么修 |
|---|---|---|
| `isv.SMS_SIGNATURE_ILLEGAL` | 签名未通过审核或不存在 | 检查 SignName 字段是否和阿里云控制台一致 · 签名状态是否「审核通过」 |
| `isv.SMS_TEMPLATE_ILLEGAL` | 模板未审核 / 模板与变量不匹配 | 检查 TemplateCode · 检查模板里 4 个变量名是否就叫 `brand/period/days/progress` |
| `isv.MOBILE_NUMBER_ILLEGAL` | 手机号格式错 | 11 位数字,无 +86 前缀,无空格 |
| `isv.OUT_OF_SERVICE` | 账户欠费 | 充值 |
| `isv.BUSINESS_LIMIT_CONTROL` | 同号码同模板 1 小时内已发过 | 等等再试 / 换手机号 |
| `isv.AMOUNT_NOT_ENOUGH` | 余额不足 | 充值 |
| `InvalidAccessKey ID.NotFound` | AccessKey ID 错或被禁用 | 检查 RAM 子账号是否启用 |
| `SignatureDoesNotMatch` | AccessKey Secret 错 | 重新粘贴 secret(注意首尾空格) |
| `IsvSmsSignBlacklist` | 签名进黑名单(发了违规内容) | 申请新签名 |

## 8. 上线后注意

- **限流**:阿里云对单手机号有限制(同模板默认 5 分钟最多 1 条 · 1 天最多 5 条)· 家庭账本天然不会触发
- **每月用量监控**:控制台 → 国内短信 → 统计分析,看月用量 · 套餐包余量
- **审核长效**:签名+模板审核通过后**长期有效**,不用每月重新报备
- **AccessKey 轮换**:建议每 6-12 个月旋转一次 RAM ak · 旋转就在 `/admin/reminders` ② 重新填,旧的 secret 会被覆盖

## 9. 如果决定后续切换 provider

V25 数据库 `family_notify_config.sms_provider` 字段已经预留 · 未来切到云片 / 腾讯只是在
`/admin/reminders` 选 provider 下拉,无需改代码。本文档届时会出对应的 yunpian/tencent 版指引。

**问题排查 · 速查**

| 现象 | 看哪 |
|---|---|
| 测试按钮点了无反应 | 检查 ② 短信渠道是否「启用」+ aksk/签名/模板都填了 |
| 一直 `isv.SMS_SIGNATURE_ILLEGAL` | 签名字段 vs 阿里云控制台签名名称必须**完全一致**(连标点都不能差) |
| `isv.SMS_TEMPLATE_ILLEGAL` 但模板已通过 | 4 变量名要严格是 `brand/period/days/progress`,大小写敏感 |
| 测试通了但日常调度没发出 | ① 模板默认 T1(实时收入+月末支出)· ② 当前期不在距截止 2 天窗口内 · ③ 该成员已提交本期完成,不在触达名单 |
| 收件人在国外 | 本服务只发国内短信 · 国外要单独申请国际短信通道(未来版本) |
