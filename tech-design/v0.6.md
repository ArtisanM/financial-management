## 6. 决策 74 · 行为体检 = 月度快照保守启发式(不要逐笔)

| 备选 | 取舍 |
|---|---|
| **A. 基于月度快照 + 净流入分解的保守启发式 · 只报强信号(选定)** | 用已有数据 · 不加录入 · 守 10min/月 |
| B. 要逐笔交易意图(买卖时点) | 违颗粒度硬约束(不到单券逐笔)· 数据没有 |
| C. 不做行为体检 | 放弃 AI 对普通人最大价值(治追涨杀跌) |

启发式(`BehaviorHeuristics` 纯函数,保守):
- **顺周期加仓("追涨")**:净流入是否集中在净值/价格高位期(相关性阈值)
- **从不止盈**:盈利大类/标的占比单调走高、长期无回撤性减仓
- **集中演变**:某大类占比逐期攀升

为什么保守:启发式有误报风险 → 只报**强信号**、文案是"信号/提醒"而非"判定对错"(承 [[feedback_llm_validator_diagnosis]] 误杀是 bug 的纪律)。无足够历史期(<6)则不报。

## 13. 实现落地与取舍记录(2026-06-05 · v0.6.0)

- **`diagnoseAssetInsight` 复用 `LlmDiagnoseService.runDiagnose`**(而非另起服务):直接复用既有 client 轮转 + 1h cache + audit + 结构化 JSON 解析,新增成本最小;以 `scope="ASSET_INSIGHT"` 在 runDiagnose 内分流到更严的 `OutputValidator.checkInsight`。一处分支,既有 `/checkup/diagnose` 行为零变更。
- **AssetInsight prompt 不含人名/账户名(隐私 by construction)**:`InsightPromptBuilder` 只接收数字 + 中性标签(类型/币种代码/桶名/锚代码),不传 `topAccountLabel`(该 label 仅用于 UI 自家展示)。因此 realNames 传空集 —— LLM 没见过名字也就无从泄露(比 family diagnose 的「脱敏映射 + 防御扫描」更彻底)。
- **Qwen 多模型兜底分类**:按官方错误码分三类 —— 免费额度用尽(429 `Throttling.AllocationQuota`/`insufficient_quota` · 403 `AllocationQuota.FreeTierOnly`)→ 标记该模型冷却 6h 并切下一模型(各模型独立额度);账户级(400 `Arrearage` · `*BillOverdue`)→ 立刻抛出由上层 failover DeepSeek(切模型无意义);其它(限流/5xx/超时)→ transient,计入短熔断后由上层换 vendor。模型列表 `K_LLM_QWEN_MODELS` 运营可配(≤10),默认首位仍 qwen-plus(语义保留)。
- **OutputValidator 扩展为 `checkInsight`(而非改 `check`)**:预测/择时禁词只在洞察路径生效;既有综合诊断的 20 条单测零回归。
- **reports 落为交叉入口、单一标的暂降级**:见 prd/v0.6.md §11 的诚实偏差说明。
