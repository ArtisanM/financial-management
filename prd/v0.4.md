## 1. 版本目标

### 1.1 中产家庭的「摸清 5 问」

家庭账房的核心是回答 5 个问题。前 4 问 v0.1-v0.3 已基本答完,第 5 问完全缺失,**v0.4 主线就是补完第 5 问 + 把前 4 问的回答更聚焦(砍重复/流水)**:

| 5 问 | 服务定位 | 当前状态 | v0.4 行动 |
|---|---|---|---|
| 1. 我家有多少钱? | 摸清 · 快照 | ✅ v0.1 net worth | 保持 |
| 2. 钱分在哪? | 摸清 · 切片 | ✅ v0.1 配置环形 + 成员饼 + 账户分布 | 去 3 处重复显示 |
| 3. 钱怎么涨/跌的? | 摸清 · 时序 | ✅ v0.1 趋势线 / v0.2 本金 vs 收益分解 | 保持(核心图) |
| 4. 风险有多大? | 摸清 · 诊断 | ✅ v0.2 风险敞口 + 流动性 + 体检规则 | 去重复(图只在一处) |
| 5. **跑赢通胀和市场了吗?** | 摸清 · 对照 | ❌ **完全缺失** | **v0.4 新答** |
| (额外)**该怎么调?** | 增值不贬值 · 决策 | ⚠️ v0.2 体检只说"偏了" | **v0.4 给具体数字** |

### 1.3 v0.4 不做什么(非范围 · 进 v0.5 排队)

| 候选 | 砍掉理由 |
|---|---|
| **录入革命**(支付宝/微信 CSV → AI 归类)| 摸清这条 v0.3 已基本到位 · 等 v0.4 决策辅助建好,再降录入成本 ROI 更高;直接做现在=工具感,价值感弱 |
| **保险账户类型** | 跟"录入革命"配套做更顺(每年 1-2 次保费录入)· v0.5 一起 |
| **税务专项附加扣除** | 政策易变 + 跟官方个税 APP 重复 · 12 月一次性 · 维护负担重 |
| **多场景账本(AA / 装修 / 旅行)** | 与"家庭账房"硬约束冲突 · v0.1 起明确非目标 |
| **二级预算 / 消费品类细化** | 与"10 分钟/月"硬约束冲突 · v0.1 起明确非目标 |
| **新增账户类型** | 现有 6 类(CASH/STOCK/WEALTH/PROPERTY/LOAN/OTHER)够用 · v0.3 已扩展持仓三态(AUTO/MANUAL/CASH) |
| **失业 N 月情景模拟** | 价值中 · 需要更完整数据基线(保险账户)· v0.5+ |

### 主线 B · 摸清第 5 问 · 跑赢通胀和市场

#### FR-61a · 净资产趋势 + CPI 实际购买力对照 · P0

> **定位**:中产最痛"我以为我赚了,实际被通胀吃掉 N%"。dashboard 净资产趋势叠一条 CPI 折算后的"实际购买力"线 → 一眼看见名义 vs 实际差距。

**模型**:
- 双线展示:
  - **名义净资产**(实线 brass-deep,当前 v0.1 行为)
  - ⭐ **实际购买力**(虚线 forest,基期为锚 deflate · 公式:`real_t = nominal_t / (1 + cpi)^(t-t0)`)
- CPI 默认 **中国 2.0%**(近 5 年央行口径均值)
- 用户可改:dashboard 顶部右上角下拉 `[CPI 2%▼]` 切换 `2% / 3% / 5%`
- 选择持久化到 `family.cpi_assumption`(新字段)

**UI**:
- 图例增 "名义" + "实际购买力(CPI N%)"
- 末点 tooltip 显示 "名义 ¥X · 实际 ¥Y(折损 Z%)"
- 颜色:名义 = 品牌色 brass-deep · 实际 = 森林绿 forest 虚线

**验收**:
- [ ] dashboard 净资产趋势图含 2 条线 + 图例
- [ ] 切换 CPI 假设 → 实际线立即重算
- [ ] tooltip 显示名义 vs 实际差额 + 折损%
- [ ] family.cpi_assumption 持久化

#### FR-61b · 账户级 XIRR vs 基准 · P0

> **定位**:reports 账户级收益表每行加"基准"列 + 跑赢/输 pill。**复用 v0.2 早就有的 `product_category.benchmark_pct` 字段(沪深 300 / 标普 500 / 中债)**,这块弹药我们一直没用起来。

**实现**:
- 每个 account 查它的 product_category_code → benchmark_pct(可能 NULL)
- 计算:`diff = account_xirr - benchmark_pct`
- pill 染色:
  - `diff > 2%` → 🟢 跑赢 +X%(forest)
  - `-2% ≤ diff ≤ 2%` → ⚫ 持平(ink-soft)
  - `diff < -2%` → 🔴 跑输 -X%(rust)
  - 未设类目 / benchmark NULL → "—"

**UI**:
- 账户级收益表加列:`基准 % | 跑赢/输`
- 表头加说明:"基准来自产品类目,可在 /admin/product-categories 微调"

**验收**:
- [ ] 每个有 benchmark_pct 的账户行显示 基准 % 和 diff pill
- [ ] 未设类目的账户显示 "—"
- [ ] pill 颜色正确(forest/ink-soft/rust)

#### FR-61c · 家庭整体跑赢/输基准 · P0

> **定位**:reports 顶部 4 KPI 区(家庭 XIRR / TWR / ...)加一行"家庭基准 = N% · 跑赢 / 输 M%"。

**算法**:
- 家庭基准 = Σ(account.endBalanceBase × product_category.benchmark_pct) / Σ(account.endBalanceBase)
- 未设类目的账户用 v0.1 默认值(STOCK→6% / WEALTH→3% / CASH→0.5% / PROPERTY→3% / LOAN→负数 mortgage_rate)
- 家庭 XIRR - 家庭基准 → 跑赢/输

**UI**:
- 家庭 XIRR KPI 卡底部加小字:`vs 基准 N% · 跑赢/输 M%`(配色同 FR-61b)

- [ ] reports 顶部 4 KPI 第一张"家庭 XIRR"下方加 vs 基准 pill

#### FR-62c · 应急金"不闲置"提示 · P0

> **定位**:复用 v0.3.3 流动性精细化数据,自动提示用户活期/工资卡有"超额闲置"应转货币基金。

**算法**:
- 当前 LIQUID 类账户余额合计(已 v0.3.3 含货币基金)
- 紧急储备需求 = `avg_monthly_expense × 应急倍数`(默认 6 个月,可改 · 复用 FR-50c 应急目标设定值)
- 阈值:`LIQUID 余额 > 应急需求 × 1.5x`(超 1.5 倍 = 闲置)
- 超额 X = `LIQUID - 应急需求`

**UI**:
- **dashboard 紧急储备 KPI 旁**:有超额时显黄色提示 banner:
  > 💡 应急金充足 + 超额 ¥X · 建议转货币基金(余额宝 / 零钱通)以获取 2%+ 年化
- 链接到 `/admin/product-categories#MONEY_FUND` 查看货币基金参考产品

**验收**:

#### FR-62d · 提前还贷 vs 投资 NPV 决策器 · P0

> **定位**:新页 `/reports/refinance` · 用户输入剩余房贷情况,系统算 "现在多还 ¥N 给房贷 vs 投资 ¥N" 的 NPV 对照,给推荐。

**输入**(form):
- 剩余房贷本金(默认从 LOAN 账户拉)
- 房贷年利率(默认从 family.mortgage_rate 取 / 用户改)
- 剩余年限(默认从房贷账户计算)
- 准备多还 ¥N(用户输入)
- 投资预期年化(默认 = 家庭 XIRR · 用户可改)

**算法**:
- 提前还贷收益 = `N × ((1+r_loan)^n - 1) / r_loan`(节省利息)
- 投资收益 = `N × ((1+r_invest)^n - 1)`(复利增值)
- diff > 0 → 推荐投资 · diff < 0 → 推荐还贷
- 边界:r_loan > r_invest → 必还(节省 = 确定 vs 投资 = 不确定)

**UI**:
- 表单(5 字段)
- 算完显双柱(还贷收益 vs 投资收益)+ 推荐
- AI 解释(可选,复用 LlmOrchestrator)

**验收**:
- [ ] /reports/refinance 200 + 表单完整
- [ ] 提交后显双柱 + 推荐
- [ ] 边界(r_loan > r_invest)推荐必还
- [ ] 默认值正确(从 LOAN 账户 + 家庭 XIRR 自动拉)

## 7. 决策追溯 · 调研 + 重定位(2026-05-13)
