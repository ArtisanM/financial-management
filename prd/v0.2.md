# 家庭账房 PRD — v0.2

> 已交付 v0.1(2026-05 阶段)。v0.2 不再扩功能广度,聚焦"用得舒服":单账户深度查看 + 移动端一站式体验(微信引导 + iOS PWA)。
>
> v0.2 全部围绕**已经存在的数据**与**已存在的页面**做增量,不引入新表(只加 `deleted_at` 软删字段);不引入新外部依赖。

---

## 1. 版本目标

家庭已经在用 v0.1 后,会自然遇到三个新问题,v0.2 集中解决:

1. "我招行卡这一年里到底进出了多少笔?" — 当前 `/entry` 只看本期、`/reports/period/{id}` 看单期、`/admin/audit` 偏运维,**没有"单账户跨期完整流水"视图**
2. "我手快填错了一笔工资,怎么撤销?" — 当前只能 reopen 周期 + 覆盖余额绕一大圈
3. "在手机微信里点开链接 → 没法用 / 太丑;每次还要在 Safari 里输 URL" — 微信内浏览器限制 + 没 PWA

v0.2 用 5 条 FR 解决,**保持节制**:不做单条流水编辑、不做 PWA 推送、不做跨账户合并视图。

## 2. 功能需求

### FR-30 · 账户详情页 `/accounts/{id}` · P0

> **核心定位修订**:不只是流水查看页,而是"**单账户的理财体检报告**"。家庭账本要让用户看到每个账户的**收益与风险表现**,不只是收支记录。

**入口**:
- 主入口:`/accounts` 列表点击账户名 → `/accounts/{id}`
- 辅入口:`/entry` 每行底部新增"全部流水 →"小字链 → 同上(deep link)
- 辅入口:`/dashboard` 账户列表段每行可点击 → 同上(v0.1 已有 sparkline 行)

**信息架构层级**(从顶部到底部):
```
1. 基础信息卡(账户名 / 类型 / 主理人 / 当前余额 / 较上期 ±)
2. 余额时间序列折线图(近 12 月)
3. 类型差异化诊断卡 ★(★ v0.2 新增,核心价值点)
   - STOCK / WEALTH → 投资视角(FR-30a)
   - CASH           → 储蓄视角(FR-30b)
   - LOAN           → 负债视角(FR-30c)
   - PROPERTY/OTHER → 简版统计(均值/极值)
4. 流水筛选条 + 时间线(按月分组,可折叠)
5. 操作底栏(返回 / 编辑 / 归档 / 导出 CSV)
```

**信息架构(从上到下)**:

```
顶部信息卡
  ├─ 账户名 · 类型 pill · 币种 · 主理人
  ├─ 当前本期余额 (大字)
  └─ 较上期 +¥X (+Y%)

中部双栏
  ├─ 左栏:余额时间序列折线图(近 12 个月)
  │     • 数据点上显示金额(沿用 v0.1 datalabels)

        • 平均月入 = avg(月度 INCOME + TRANSFER_IN)
        • 平均月出 = avg(月度 EXPENSE + TRANSFER_OUT)

筛选条
```

### FR-30a · 投资诊断卡(STOCK / WEALTH)· P0

**位置**:基础信息卡 + 时间序列折线图之下,流水筛选条之上。

**布局**(双栏 5 + 5):

```
左栏:收益指标
  ┌─────────────────────────────────────────┐
  │ 累计收益       +¥48,200    +12.4%      │
  │ 累计净流入     ¥390,000                │
  │ 当前价值       ¥438,200                │
  │ ──                                      │
  │ 年化 XIRR      6.8% (近 24 月)         │
  │ 年化 TWR       7.1% (剔除资金进出)     │
  │ ──                                      │
  │ 净值曲线 (起点 1.000)  ╭──╲╱──╮        │
  │                       ╱       ╲        │
  └─────────────────────────────────────────┘

右栏:风险指标
  ┌─────────────────────────────────────────┐
  │ 最大回撤       -8.3%   于 2025-10      │
  │ 月波动率(年化) σ = 11.2%                │
  │ Sharpe 比率    0.61                     │
  │ 回撤恢复       42 天                    │
  │ ──                                      │
  │ 月收益分布柱状(12 月)                  │
  │  ┃▌▍▍▎▏ ▏▎▍▌▋▊▉                       │
  │  -3 -2 -1  0 +1 +2 +3 (单位 %)         │
  └─────────────────────────────────────────┘
```

**计算口径**:
- **当前价值** = 本期 period_snapshot.end_balance(显示币种)
- **累计净流入** = Σ(INCOME + TRANSFER_IN − EXPENSE − TRANSFER_OUT) 全部历史
- **累计收益** = 当前价值 − 累计净流入
- **累计收益率** = 累计收益 / 累计净流入(若分母为 0,显示 "—")
- **年化 XIRR** = `XirrCalculator.annualizedOrCumulative(account-level cash flows + 期末市值)`,复用 v0.1 已有
- **年化 TWR** = 月度切片 TWR `Π(1 + r_i) − 1`,复用 v0.1 `TwrCalculator`
- **净值曲线** = 月度切片连乘:nav(month i) = nav(i-1) × (1 + r_i),起点 1.000;在折线图上表现为"剔除资金进出后的纯收益曲线"
- **最大回撤** = 月颗粒。`MaxDrawdownCalculator.compute(navSeries)`,新增 calc 类
- **月波动率(年化)** = std(月收益率) × √12
- **Sharpe** = 年化收益率 / 年化波动率(无风险率取 0%,简化口径,标黄"简化口径无风险率=0%")
- **回撤恢复天数** = 最大回撤底部到下次创新高的月份差 × 30(近似)
- **月收益分布** = 近 12 月的月收益率列表,柱状图

**验收**:
- [ ] 仅 STOCK / WEALTH 显示
- [ ] 7 个指标全部计算正确(单元测试覆盖)
- [ ] 净值曲线起点严格 1.000
- [ ] 最大回撤 hover 显示"于 YYYY-MM 触底,回撤 X%,N 天后恢复"
- [ ] 数据点 < 6 个月时,Sharpe / σ 显示"—"+ 角标"样本不足"

### FR-30b · 储蓄诊断卡(CASH)· P0

**布局**(单栏 4 个 KPI 大数 + 月度收支柱):

```
┌─────────────────────────────────────────────────┐
│ 紧急缓冲月数    ⏱ 14.2 月  (健康 ≥ 6)            │
│ 平均月入        ¥34,500                          │
│ 平均月出        ¥21,800                          │
│ 月均净储蓄      +¥12,700  (净储蓄率 36.8%)       │
│ ──                                              │
│ 月度收支柱状(12 月)                            │
│  ┃▌▍▌▎▏▎▍▌▋▊▉                                │
│   ↑ 收入    ↓ 支出    · 净储蓄折线              │
└─────────────────────────────────────────────────┘
```

**计算口径**:
- **紧急缓冲月数** = 当前余额 / 月均消费(本账户 EXPENSE,仅算近 12 月)
- **平均月入 / 月出** = 近 12 月的 INCOME / EXPENSE 平均
- **月均净储蓄** = 平均月入 − 平均月出
- **月度收支柱状** = 12 月的 income / expense / net 三系列

**风险信号**(banner 警示):
- 紧急缓冲 < 3 月 → 红色 banner "缓冲不足 3 月,建议留 6 月以上"
- 单月最大支出 > 平均月出 × 3 → 黄色 banner "最近某月异常大额支出"
- 连续 3 月净储蓄 < 0 → 红色 banner "近 3 月持续超支"

**验收**:
- [ ] 仅 CASH 显示
- [ ] 4 个 KPI 数字正确
- [ ] 风险 banner 触发条件准确

### FR-30c · 负债诊断卡(LOAN)· P0

**布局**:

```
┌─────────────────────────────────────────────────┐
│ 剩余本金        ¥780,000   (原始 ¥1,000,000)    │
│ 已还本金        ¥220,000   ▰▰▰▰▰▰▰▱▱▱ 22%      │
│ ──                                              │
│ 本期还款        ¥7,800                          │
│ 月均还款        ¥6,500  (近 12 月)              │
│ ──                                              │
│ 预计还清        2034 年 03 月  (按当前速度)     │
│ 剩余月数        100 月 ≈ 8.3 年                 │
│ ──                                              │
│ 累计利息        ¥45,000  (若已分类登记)         │
│ 还款历史柱状(12 月)                            │
└─────────────────────────────────────────────────┘
```

**计算口径**:
- **原始本金** = 该 LOAN 账户**最早一期** snapshot.end_balance 绝对值
- **剩余本金** = 当前期 snapshot.end_balance 绝对值
- **已还本金** = 原始 − 剩余
- **本期还款** = 本期 transfer_in(因为还款是从其它账户 transfer 进 LOAN)
- **月均还款** = 近 12 月 transfer_in 平均
- **预计还清月数** = 剩余本金 / 月均还款,向上取整
- **预计还清日期** = 当前月 + 剩余月数
- **累计利息** = Σ(本期 EXPENSE 类别=利息),若用户从未登记则显示 "—"

**风险信号**:
- 月均还款 < 上期变化量(大概率拖延)→ 黄色 banner "近期还款速度放缓"
- 信用卡类账户(子类型,v0.3 加)余额持续上升 → 红色 banner

**验收**:
- [ ] 仅 LOAN 显示
- [ ] 进度条 + 还清月数计算正确
- [ ] 累计利息字段在用户未登记时优雅显示 "—"

### FR-30e · 智能建议 / Action Items · P0

> **产品愿景对齐**:家庭账本不只是"看数据",而是"**给建议**"。普通家庭(尤其非技术成员)不会自己消化 Sharpe/σ/回撤这种术语,他们要的是 **"我现在该做什么"**。这是 v0.2 区别于"账户列表/流水查看器"的核心价值。

**位置**:在差异化诊断卡之后、流水时间线之前。**最显眼**——用户进入账户详情页第一个看到结论,然后才看明细。

**形态**(纸质便签卡片堆叠,每条建议一张):

```
┌────────────────────────────────────────────────────────┐
│ 📌 来 · 自 · 账 · 房 · 的 · 三 · 条 · 提 · 醒          │
│                                                        │
│ ┌──────────────────────────────────────────────────┐  │
│ │ ⚠️ 缓冲过厚                          〔流动性建议〕│  │
│ │                                                  │  │
│ │ 招行储蓄卡-工资 当前余额 ¥73,200,                │  │
│ │ 相当于 14 个月的家庭支出 — 已超过推荐的 6-9 月。  │  │
│ │                                                  │  │
│ │ 💡 建议:把 ¥35,000 拨到「蚂蚁财富-基金」(年化   │  │
│ │    3.5% 的稳健理财),预计每年多挣 ¥1,225。       │  │
│ │                                                  │  │
│ │   [→ 现在划转]   [✕ 这条不适用]                  │  │
│ └──────────────────────────────────────────────────┘  │
│                                                        │
│ ┌──────────────────────────────────────────────────┐  │
│ │ ✓ 表现良好                          〔收益建议〕│  │
│ │                                                  │  │
│ │ 该账户近 12 月年化 6.8%,跑赢家庭其它理财账户    │  │
│ │ 平均(5.2%)— 这是一笔不错的资产。               │  │
│ │                                                  │  │
│ │ 💡 建议:可适度加大配置比例,但留意风险刻度的     │  │
│ │    最大回撤 −8.3%,做好下跌承受准备。            │  │
│ └──────────────────────────────────────────────────┘  │
│                                                        │
│ ┌──────────────────────────────────────────────────┐  │
│ │ 🔔 提醒                              〔进度建议〕│  │
│ │                                                  │  │

│ │ 速度略慢,按当前节奏还清日从 2034-01 推迟到       │  │
│ │ 2034-03。                                        │  │
│ │                                                  │  │
│ │ 💡 建议:不影响生活的前提下,本月多还 ¥800。     │  │
│ └──────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘
```

**生成机制**:**规则引擎**(非 LLM、非 AI 模型,纯 Java 规则)。每个账户在用户访问详情页时,实时跑一组规则(< 50ms),命中即生成 1 条建议。每条建议含:

```java
record AccountAdvice(
    AdviceLevel level,       // OK / INFO / WARN / DANGER
    AdviceCategory category, // LIQUIDITY / EFFICIENCY / RETURN_QUALITY / CONCENTRATION / PROGRESS / BEHAVIOR
    String headline,         // "缓冲过厚"
    String body,             // 完整描述含数字
    String suggestion,       // "建议把 ¥35,000 拨到 ..."
    Optional<Action> action  // 一键操作(可选,如"现在划转")
) {}
```

**v0.2 内置规则清单**(每个 ~20 行 Java,共 ~10 条):

| Rule ID | 触发条件 | 适用账户 | Level |
|---|---|---|---|
| `LIQ-1` | 紧急储备月数 < 3 | CASH | DANGER |
| `LIQ-2` | 紧急储备月数 < 6 | CASH | WARN |
| `EFF-1` | CASH 余额 > 月均支出 × 12 | CASH | INFO |
| `EFF-2` | WEALTH/STOCK 年化 < 银行活期(0.35%) | STOCK/WEALTH | WARN |
| `RET-1` | 年化 < 4% & Sharpe < 0.3 | STOCK/WEALTH | INFO |
| `RET-2` | 年化 > 8% & Sharpe > 0.8 | STOCK/WEALTH | OK(表扬) |
| `CON-1` | 单账户 > 总资产 40% | 全部 | INFO |
| `PRG-1` | LOAN 月均还款 < 上期还款 × 0.85 | LOAN | WARN |
| `PRG-2` | LOAN 月均还款 > 上期还款 × 1.20 | LOAN | OK(表扬) |
| `BHV-1` | 信用卡(LOAN 子类型,v0.3)欠款连涨 3 月 | LOAN | DANGER |

每条规则的 `body` / `suggestion` 用 String.format 填入具体数字,**不用 LLM**,完全可预测。

**信号设计**:
- `OK`:绿色描边 + ✓ 图标 + 表扬语气("这是一笔不错的资产")
- `INFO`:灰色描边 + 💡 + 中性建议
- `WARN`:黄铜色描边 + 🔔 + 提醒语气
- `DANGER`:朱红色描边 + ⚠️ + 强烈语气

**建议数量**:每个账户最多展示 3 条(按 level DANGER > WARN > INFO > OK 排序);如果什么规则都不命中,显示一条 "✓ 此账户暂无需特别关注" 占位。

**用户操作**:
- "→ 现在划转" / "→ 看其它账户" — 跳到对应页面或弹划转 modal
- "✕ 这条不适用" — 软隐藏(localStorage 记 `dismissed_advice_{accountId}_{ruleId}`),30 天内本规则不再触发

**验收**:
- [ ] 至少命中 1 条规则的账户进入页面,看到建议块
- [ ] 数字与诊断卡一致(同一计算口径)
- [ ] DANGER / WARN 排在 INFO / OK 之前
- [ ] dismiss 后 30 天内不再显示
- [ ] 单账户最多 3 条
- [ ] **加规则不需要改前端模板**(规则在 service 层,模板渲染 List<AccountAdvice>)

### FR-30f · 产品类目 + 风险评级 + 基准对照 · P0

> **核心问题**:`XIRR 6.8%` 这个数字单独看没意义。是好是坏?要看它对照什么 — 一只 A 股基金 6.8% 是跑输沪深 300,一只货币基金 6.8% 是异常高。**没有基准就没有判断**。
>
> 这是 v0.2 的"诊断"和"建议"两个功能能落地的前提:每个账户必须先归属一个**产品类目**(货币基金 / A 股 / 美股 / 房产 ...),由类目确定风险等级 + 基准年化,后续诊断卡和智能建议都用这个基准做对比。

**新增数据**:`product_category` 参考表(系统预置 ~14 个类目)+ `account.product_category_code` 字段。

**预置类目清单**(初版,管理员可在 `/admin/product-categories` 微调):

| code | 中文名 | 风险等级 | 基准指数 / 描述 | 长期年化(基准) | 适用 type |
|---|---|---|---|---|---|
| `CASH_DEPOSIT` | 现金活期 / 工资卡 | ★ 极低 | 银行活期利率 | 0.35% | CASH |
| `MONEY_FUND` | 货币基金(余额宝类) | ★ 极低 | 7 日年化均值 | 2.0% | CASH / WEALTH |
| `BANK_WEALTH` | 银行理财(R2 稳健) | ★★ 低 | 同业基准 | 3.2% | WEALTH |

### FR-32 · 单条流水软删除 · P0

**触发**:每条流水末尾 `⋮` 菜单 → "删除此条"(二次 confirm)

**约束**:
- 周期 status=CLOSED 的流水**不可删**(POST 直接 toast "周期已关闭,需先重新打开")
- LOAN 账户的"自动 prefill 草稿 transfer"**不可删**(因为是系统生成的,改 LOAN 余额时再处理)

**端点**:
- `POST /entry/cash-flow/{cfId}/delete` — 软删 cash_flow
- `POST /entry/transfer/{tId}/delete` — 软删 transfer
- `POST /entry/snapshot/{periodId}/{accountId}/delete` — **不支持**(snapshot 是余额权威,只能 reopen 周期后重填)

**数据模型**:
```sql
ALTER TABLE cash_flow ADD COLUMN deleted_at TIMESTAMP(3) NULL;
ALTER TABLE transfer  ADD COLUMN deleted_at TIMESTAMP(3) NULL;
CREATE INDEX idx_cash_flow_active ON cash_flow(period_id, account_id, deleted_at);
CREATE INDEX idx_transfer_active ON transfer(period_id, deleted_at);
```

**所有既有 mapper 查询**加 `WHERE deleted_at IS NULL` 过滤。

**软删后的副作用**:
- snapshot.end_balance **不动**(用户手动决定的权威值)
- 由此产生的差额会出现在该账户的"未解释金额"上 → 用户看到后可手动校准
- 审计:`audit_log type=CASH_FLOW_DELETE / TRANSFER_DELETE`,含 `payload_json`(流水原值)

**验收**:
- [ ] 删除 OPEN 期流水成功;DB 中 `deleted_at` 写入,UI 该条消失
- [ ] 删除 CLOSED 期流水触发 toast 拒
- [ ] 删除后:本账户"收入/支出/转入/转出"4 字段汇总变化、未解释金额相应增加 / 减少
- [ ] 审计日志含被删除流水的 amount/kind/note,可恢复
