# PRD v0.10 · 仪表盘实时「人赚 vs 钱赚」拆解 + 实时收支趋势

> 状态:**待评审(PRD + preview 阶段)**。preview:`preview/v0.10/dashboard-cashflow.html`。
> 本版只补「实时收支」这半边,不碰已有 5 KPI、不恢复 v0.4 砍到 /reports 的月均/双柱大图。

## 背景与动机

家庭账房三支柱之一是**记账**:每月花 10 分钟把各账户余额、收入、支出填一遍。但仪表盘(实时视图)目前**只显示资产/投资侧**,收入与支出一个实时数字都没有 ——

- v0.4(FR-60a)做过「KPI 收敛 9→5」,把**月均收入/支出/储蓄率/收支双柱图**搬去了 `/reports` 储蓄区。当时对(首页太挤、报表版更专业)。
- 但 `/reports` 锚定的是**「最近已关账账期」快照**(v0.5.5 FR-94),**故意不含进行中的本月**。结果:**本月的收入/支出/净流入,在全站任何"实时"位置都看不到**。
- 更刺眼:仪表盘 KPI「本月资产收益」的算法是「(期末 − 期初 − **本月净流入**) ÷ 期初」,副标题写着「**剔除收入**」——它算了收入又减掉,却从不显示收入本身。用户自然问:**那收入呢?**

概念上,app 的主轴是 **净资产变化 ΔNW = 人赚(收入−支出的净流入)+ 钱赚(投资损益)**。现在实时仪表盘上**钱赚那半边富得流油,人赚那半边全黑**。本版补齐人赚那半边。

> 关键事实(降低实现风险):三个核心数**已现成**——`KpiSnapshot.netWorthDelta`(ΔNW)、`lastNetInflow`(人赚 · v0.5.3 已按 viewCurrency 换算)、`monthlyPnlAmount`(钱赚)。拆解卡主要是展示层。

## 功能需求(FR)

### FR-165 · 仪表盘新增「本期:人赚 vs 钱赚」实时拆解卡
- 位置:5-KPI 带之后、净资产趋势(`dash-trend`)之前,新 `<section id="dash-cashflow">`(实时锚点 = 当前 OPEN 账期)。
- 头部:**本期净资产变化 ΔNW**(大字,正负染色);右上角 pill:`本期 YYYY-MM · 进行中` + `已填报 N / M 个账户`。
- **占比条**:一条水平堆叠条,人赚段(forest)/ 钱赚段(brass-deep)按金额占比分宽,百分比浮在段上。
- **两张子卡**:
  - **人赚 · 你主动存下的** = 本期净流入;明细「本期收入 ¥X − 支出 ¥Y」;一句话「你能控制的那部分」。
  - **钱赚 · 钱替你赚的** = 投资损益(monthlyPnlAmount);一句话「市场给的那部分」。
- 脚注:`ΔNW = 人赚 + 钱赚`,与「本月资产收益」同源(那个 KPI 剔除的"收入"就是这里的人赚)。

### FR-166 · 本期收入/支出明细 + 完整度诚实标注
- 收支是**选填**(填报页底部汇总「留空=未填」)。拆解卡**必须**带完整度,绝不拿半截数据当全貌:
  - **空态**(本期 0 填):整卡换成引导态 —— 「本期还没填收支 · 去填报页填一次(~15 秒),系统自动拆人赚/钱赚」+「去填报 →」按钮。**钱赚侧若净资产差额已可算,仍可单独显示**(它不依赖收支填报)。
  - **半填态**(0 < 已填 < 全部):正常显示,但挂琥珀色 pill「已填 N/M · 收支可能不全」,并注明「人赚是**下限**(只统计已填账户),补全后通常会变;钱赚不受影响」。
  - **完整态**(全填):正常完整展示。
- 复用现有 `cashflowFilled / cashflowTotal`(controller 已在算)。

### FR-167 · 实时收支趋势小图(含进行中本月)
- 紧随拆解卡的紧凑图:近 6~12 期的**收入(forest 柱)/ 支出(rust 柱)/ 净流入=人赚(brass-deep 线)**。
- **含进行中的本月**(最右一期浅色/描边区分,标「进行中」)——这正是它与 `/reports` 储蓄区的区别(后者只到上一已关账期)。
- 数据所有数值用 **datalabels 浮在柱顶/数据点上**(不靠 hover tooltip)。
- 角落给一条「看完整收支分析(月均 / 储蓄率 / 储蓄能力中位)→ /reports#savings」的去向链接,不在仪表盘重做那套深度分析。

### FR-168 · 与既有 KPI 呼应 + 可隐藏
- 「本月资产收益 · 剔除收入」KPI 文案微调,指向本卡(让"被剔除的收入"可达)。
- 本卡作为一个**指标偏好项**(family metric pref,如 `cashflow_split`),可在「指标设置」里开关;**默认开**(它是补主轴空白,不是边角料)。

## 验收基线(v0.10)

1. 走用户实际点击路径(顶层进首页):看到「人赚 vs 钱赚」拆解卡 + 实时收支趋势,**桌面 + 手机**两版都对齐不破版。
2. **三态都对**:完整态显三数;空态显引导 + (可算时)钱赚;半填态显琥珀完整度 pill + 人赚为下限的说明。
3. **币种恒等**:CNY/USD/HKD 三视图下 `ΔNW = 人赚 + 钱赚` 完全成立,金额按 fx 缩放、占比不变 → 扩 `CurrencyInvarianceTest` 护栏(承币种不变性纪律)。
4. 趋势图含进行中本月、数值 datalabels 浮于数据上;`/reports` 储蓄区不受影响、仍是已关账快照。
5. 同步 dashboard **长文目录**(`_toc` + `toc.js` + 新 section `id`);qa-run 加 `v10-CASHFLOW-*` 守护(卡在、三态、datalabels、TOC 锚点)。
6. mvn 全绿 + 文档同步 prd/tech-design/qa-cases/CHANGELOG/README;纯展示 + 轻量 service,零 schema、向后兼容。

## 非目标(明确不做)

- **不恢复** v0.4 砍掉的月均收入/支出、储蓄率中位、收支双柱大图 —— 那套深度分析留在 `/reports` 储蓄区(已关账、更专业)。
- **不做分类预算 / 记账明细**(那会跑偏成通用记账 app,违三支柱聚焦)。
- 不改 `/reports` 的已关账快照语义;不动现有 5 个 KPI 的算法。
