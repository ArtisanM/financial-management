# PRD · v0.9 · 根路径公开落地页(降钓鱼误判 + 对外门面)

> 起因:Chrome 把 `dixi-token.top` 报「欺骗性网站(Deceptive site)」。排查确认**非被黑、非证书问题、Google 静态库也干净**——是 Chrome 实时反钓鱼模型的误判,命中模式是「低信誉 .top 域 + 域名含 token + 根路径首屏即裸登录框」。本版做「治标且本就该做」的一件事:**给根路径 `/` 一个公开介绍落地页**,把裸登录降为次级入口。既降误判信号(配合 Safe Browsing 申诉),也给项目一个像样的对外门面(README / V2EX / Demo 链接点进来不再是个光秃秃的登录框)。
>
> 诊断已核对真实代码:`/` 无公开映射 → Security 拦截未登录请求 → 302 跳 `/login`(`SecurityConfig.java:93` permitAll 清单无 `/`);登录页本身干净(`<title>登录 · 家庭账房`,无注入)。

## 0. 背景

同一套代码同时跑在 prod(`dixi-token.top`,真实私有数据)和 beta(`beta.dixi-token.top`,公开合成 Demo)。所以落地页文案必须**中性**:既适合"这是我自托管的私有账房"的拥有者,也适合"这是个可试用的开源 Demo"的陌生访客——**不在落地页硬编码 Demo 账号**(Demo 账号只放 README / 宣发渠道,避免 prod 真实实例暴露登录提示)。

## 1. 目标用户 & 场景

- 陌生访客(从 README / V2EX / 搜索点进来):落地 `/` 第一眼看懂"这是什么、给谁、开源、怎么进",而不是被一个登录框吓退或被浏览器标红。
- 站点拥有者 / 已登录用户:访问 `/` 不被打扰,直接进账房(`/dashboard`)。

## 2. 非目标(本版不做)

- 不做营销官网级别的多页站点 / 博客 / 定价页(本就免费开源)。
- 不改登录逻辑、认证、会话(`/login` 行为完全不变)。
- 不引入 i18n 框架(中文为主,英文最多点缀)。
- 不动任何数据模型 / schema / 后端计算。
- 不在 prod 落地页放 Demo 账号或"点此试用"诱导(那是 README / beta 的事)。

## 3. 功能需求

### FR-160 · 根路径分流(公开落地页 vs 进账房)

- **未登录** GET `/` → 返回 **200 公开落地页**(不再 302 跳 `/login`)。
- **已登录** GET `/` → 302 跳 `/dashboard`(老用户无感,不强迫看落地页)。
- `/` 加入 Security `permitAll`;`/login` 及其它鉴权行为不变。
- **现状**:`/` 未放行 → 未登录一律跳裸登录页。**验收**:无 Cookie 访问 `/` 得 200 且是介绍页;带有效会话访问 `/` 跳 dashboard。

### FR-161 · 落地页内容(对外门面 · 复用 README 口径)

一屏到底的单页,自上而下:

1. **品牌 + 一句话定位**:`家庭账房 / Family Ledger` + tagline「每月 10 分钟,把全家散在各处的钱,变成一张只在你自己服务器上的资产全局图」。
2. **三支柱**(承 [[feedback_promo_three_pillars]],不跑偏):① 月度快照记账 ② 真实年化收益(XIRR/TWR,人赚 vs 钱赚)③ AI 持仓诊断(只解读,不荐产品、不预测涨跌)。
3. **功能总览截图**:用已有的 `feature_summary_total.jpg`(桌面 + 移动一图)。
4. **适合 / 不适合**:复用 README 的 ✓/✗ 两栏(能自架 Linux / 在意隐私 / 想要真实年化 ↔ 想商店下 App / 想逐笔流水账)。
5. **自托管 · 隐私**:数据只在你自己服务器、真名脱敏后才喂 LLM、Apache-2.0。
6. **CTA**:主按钮「**登录**」(→ `/login`)+ 次按钮「**GitHub 源码**」(完整 URL 醒目,承 [[feedback_promo_github_prominent]])。
- 风格:晚清账册风(Fraunces / Noto Serif SC / JetBrains Mono + paper/ink/brass 调色),**全 inline SVG,无 emoji**(承 [[feedback_no_emoji]]);文案面向非技术家人也看得懂(承 [[feedback_user_friendly_naming]])。
- **验收**:六块齐全;截图能加载;GitHub 为完整 URL;无 emoji;手机端单列不破版。

### FR-162 · 降钓鱼误判信号(配合申诉)

- 根路径首屏是**有品牌、有说明、有开源链接的真实内容页**,而非裸登录框 —— 直接削弱 Chrome「deceptive login page」的判定特征。
- 落地页含指向 GitHub 开源仓库的可见链接(可审计 = 非钓鱼的有力信号)。
- **验收**:`curl` 未登录 `/` 的 HTML 含项目说明文案 + GitHub 完整 URL + 截图引用;不再是 `redirect:/login`。

## 4. 验收基线

1. 无 Cookie GET `/` → 200,渲染落地页(六块齐全、截图在、GitHub 全 URL、无 emoji、手机单列)。
2. 已登录 GET `/` → 302 `/dashboard`;`/login` 行为不变;退出后再访问 `/` 又是落地页。
3. 不回归:dashboard / reports / 各鉴权页对未登录仍要求登录(`permitAll` 只加了 `/`,没放开别的)。
4. 新增黑盒守护:`v09-LAND-*`(未登录 `/`=200 含说明 + GitHub;已登录 `/`→dashboard;首屏非裸登录)。
5. 文档同步 prd/tech-design/qa-cases/CHANGELOG;mvn test + qa-run 全绿;beta 真机验收。
