## 1. 版本目标

家庭用了 v0.2 几个月后,会自然遇到三个新问题,v0.3 集中解决:

1. **"我们家到底在朝什么财务目标走?差多少?还要几年?"** — v0.2 看趋势/配置/体检,但没有"目标 + 进度 + 预测"这个轴,长期视角缺失
2. **"AI 只在体检页冒一次头,其它地方都还是死规则"** — v0.2 的 LLM 综合诊断只覆盖资产体检场景,目标设定 / 月报 / 预警 全没接 AI
3. **"股票账户每月填余额很烦,要打开券商 app 看市值再回来填"** — v0.2 颗粒度是账户聚合数,价格变化全靠人工反映

v0.3 由三条主线 + 一个横切组成:

**主线 A · 财务目标体系**(FR-50 系列):
- FR-50 目标主页 `/goals` + 三情景预测
- FR-50a 退休 / FIRE 目标
- FR-50b 子女教育金目标
- FR-50c 应急储备目标
- FR-50d Dashboard 顶部"目标进度条带"(信息架构 C 混合)

**主线 B · 储蓄能力指标**(FR-51 系列):
- FR-51 填报页 **by 成员** +2 框(2026-05-13 修订:每个家庭成员各自填自己的"本月总收入/支出";家庭总额 = SUM 跨成员)
- FR-51a 储蓄率 + 月储蓄能力 + 反推月供 指标(所有"月均"都用 SUM(成员) / 期数)
- FR-51b 替换 v0.2 既有"空架子"图(桑基图 / 收支比例环形 → 月度收支双柱)

**主线 C · 股票账户自动估值**(FR-52 系列):
- FR-52 持仓管理页 `/accounts/{id}/holdings` + 持仓级 AUTO/MANUAL 混合模式
- FR-52a 新浪/腾讯双源拉价 + 圆形熔断
- FR-52b /entry 持仓变动追踪
- FR-52c 三市场覆盖:美股 + A 股 + 港股
- FR-52d **多币种混合持仓**(2026-05-13 修订):一个 STOCK 账户内可同时放 HKD/USD/CNY 持仓 ·
  各持仓按原币种算市值(BABA USD × USD→HKD)→ 折算到账户币种汇总 →
  写入 account_balance。fx 自动经家庭本位币中转(USD→CNY→HKD)。
  MANUAL 持仓 manual_value 是账户币种值(不做汇率换算)。

**横切关注 D · AI 介入扩展**(FR-53 系列 · 复用 v0.2 LlmOrchestrator):
- FR-53a 目标设定向导 AI(推荐合理默认值)
- FR-53b 目标页月报叙事 AI
- FR-53c 偏离预警 + 建议 AI
- FR-53d 体检页诊断增强(扩展 v0.2 现有 prompt)

**关注点分离原则** ★:
- 账户详情页 `/accounts/{id}` 仍只看账本(v0.2 不动)
- 资产体检 `/checkup` 仍只看诊断(v0.2 不动,prompt 微扩展)
- 目标 `/goals` 只看长期视角(进度 · 三情景 · 月报)
- Dashboard 顶部加目标条带作为入口,主体净资产/趋势/配置/账户分布/成员分布 **完全保留**

## 2. 非范围(本版本不做)

| 砍掉的 | 理由 |
|---|---|
| 一次性目标(还贷 / 大额采购) | 颗粒度浅 · v0.4 加 |
| 蒙特卡洛 / 历史回放预测 | 参数复杂,可视化不直观,违反"妻子能看懂"原则 |
| 支出结构分析 / 类目化流水 | 需用户分类每笔流水 · 冲击 10min 约束 |
| 桑基图 + 收入支出比例环形图 | v0.2 既有"空架子"(从无数据源) · 本版删除替换 |
| 基金 / ETF 自动估值 | 场内场外数据源完全不同 · 延后 v0.3.x |
| 加密货币自动估值 | 同上 |
| 单股 / 单券持仓的 XIRR | 现有账户级 XIRR 已够用 |
| 目标账户绑定(多对多 / 类型映射) | 用全资产口径 → 零设置 |
| 行业基准对比(同年龄段中位数) | 数据源不可信,做出来会误导 |
| 节假日表手动维护 | 用"拉价失败 → 沿用 T-1"简化 · 无需交易日历 |

### FR-50c · 应急储备目标 · P0

> **定位**:目标 = N 个月月均开支 · 进度 = 现金类账户余额(不是全资产)/ 目标。
> **特殊**:此目标的进度口径**与其他目标不同** — 仅算 CASH 类型,因为应急储备的意义是"流动性"。
> **更特殊**:目标值是 derived 而非用户输入 — 用过去 3 期 total_expense_input 中位数 × N。

**入口**:`/goals/new/emergency` 或 `/goals` → [+ 新建目标] → 应急储备

**信息架构 · 创建向导**:

```
═══════════════════════════════════════════════════════
新建目标 · 应急储备
═══════════════════════════════════════════════════════
基础参数
  储备月数       [6] 个月(默认 6 · 行业惯例 3-12)

基线选择
  ○ 自动基线(过去 3 期 total_expense_input 中位 = ¥18,000/月)
  ○ 手填基线 [_______] ¥/月

派生(只读)
  目标储备金额 = baseline × months_target = ¥108,000
  当前进度    = SUM(CASH 类账户余额) / 目标 = 85%
  状态        ✅ 已超过最低储备线(若进度 ≥ 100% 显此 badge)
              ⚠ 低于储备线 50%(若进度 < 50% 显警告)
```

**验收**:
- [ ] 月数 default 6 · validator 1-24
- [ ] 自动基线计算正确:从 `period.total_expense_input` 非 NULL 的过去 3 期取中位
- [ ] 过去 3 期都 NULL → 自动基线灰显 + 提示"先去 /entry 补录月度收支"
- [ ] 手填基线 validator 100-100k
- [ ] 进度仅算 CASH 类账户余额(不含 WEALTH / STOCK · 流动性硬约束)
- [ ] 进度 ≥ 100% 显 ✅ badge · < 50% 显 ⚠ 警告
