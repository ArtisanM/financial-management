/ 智 · 能 · 建 · 议 · 产 · 品 · 决 · 策

AI 介入设计 · 60% 规则 / 40% LLM / 0 幻觉

资产体检的智能建议不是纯模板填空,也不是纯 LLM 生成。规则引擎给硬骨架(数字 + level + 触发逻辑),LLM 给语言层(自然、专业、克制的投资顾问语调),OutputValidator 双重锁数字幻觉。 本页可视化展示这套架构的产品决策。

/ 文 · 案 · 对 · 照 · 中 · 式 · 古 · 典 · vs · 现 · 代 · 投 · 顾

语调选择

视觉品牌(账房 logo / 印章 / 印泥红 / "印泥洒了"错误页)保留中式雅致 — 这是品牌资产。 但 AI 输出文案改用现代专业投资顾问 — 否则妻子(非技术)看不懂"师傅打理 / 挪点钱"这种家常表述,工程师(自己)觉得不专业。

场景 · 流动性过剩(WARN)· 同样的 hardFacts
✗ 中 · 式 · 古 · 典(否决)

"你这账户压着 14 个月家用,搁着也是搁着。挪 ¥35k 到货币基金,留 5 月应急,一年多挣 ¥1,225。"

✓ 现 · 代 · 投 · 顾(选定)

流动性月数 14.2,显著高于推荐区间(6-9 月)。建议将 ¥35,000 配置至货币基金,保持 5 月应急储备,预计年化增收 ¥1,225。

场景 · 跑输基准(WARN)
✗ 中 · 式 · 古 · 典

"你这美股账户最近 12 个月赚了 9.2%,跑得比 QQQ 慢点(10.5%),要不直接买点 QQQ ETF?"

✓ 现 · 代 · 投 · 顾

近 12 月年化 9.2%,跑输基准 QQQ(10.5%)1.3 个百分点。考虑配置 30% 仓位至 QQQ ETF 直接跟踪基准,降低主动选股偏差。

场景 · 集中度告警(DANGER)
✗ 中 · 式 · 古 · 典

"这一个账户占了你家 42% 的资产,有点扎堆,挪点出去到债券基金更稳。"

✓ 现 · 代 · 投 · 顾

单账户敞口 42%,超过推荐集中度上限(30%)。建议将 ¥80,000 再配置至债券基金,预计组合波动率 σ 由 11.2% 降至 8.4%。

禁 · 词(LLM 输出层)

师傅 · 打理 · 挪 · 留个 · 家底 · 搁着 · 扎堆 · 给你算算

这些词在品牌 chrome 用,不进 AI 建议正文

必 · 用 · 术 · 语

流动性配置 · 应急储备 · 风险敞口 · 配置比例 · 再平衡 · 年化 · 跑输/跑赢 · 跟踪误差 · 加速偿还 · 利息成本

/ 失 · 败 · 兜 · 底 · 用 · 户 · 无 · 感

3 种失败 → 3 种降级

失 败 1 · API 超时 / 502 / 限流

LLM API 调用失败、限流(QPS 满)、网络故障

→ 降级

显示 templateText(规则填空版)+ ⚠ 角标"模板"

失 败 2 · 幻 觉 数字异常

LLM 输出含 hardFacts 白名单外的数字 / 百分比

→ 降级

Validator 拦截 → 回退 templateText + 写 audit_log + 记录 prompt 用于改进

失 败 3 · 禁 词 出现产品名

LLM 输出含具体产品名(余额宝 / 510300 / 茅台等)

→ 降级

Validator 扫描禁词命中 → 回退 templateText + audit_log

用户感受:任何失败场景都看到一段中文建议(模板版),不会看到错误信息 / 空白 / 转圈卡死。 所有失败案例进 audit_log 后台分析改进 prompt,5% 降级率内可接受。
