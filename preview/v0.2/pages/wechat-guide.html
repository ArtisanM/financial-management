微信内打开 · 引导跳浏览器

微信内置浏览器对 PWA / Service Worker / Local Storage / iOS Safari 部分 API 支持不全, 家庭账房在被微信打开时会显示一层引导遮罩,提示家庭成员在系统浏览器(Safari / Chrome)中打开。 非微信环境(Safari / Chrome / Firefox 桌面)不出现。

在浏览器中打开

微信内浏览器对账房功能支持不全,建议改在 Safari / Chrome 等浏览器中打开,可获得完整体验。

  1. 1 点击右上角的 ⋯ 按钮
  2. 2 选择 「在浏览器中打开」
24 小时内不再提示

UA 检测

// templates/fragments/layout.html
const isWeChat = /MicroMessenger/i.test(navigator.userAgent);
const dismissed = localStorage.getItem('wechat_dismissed');
const expired = !dismissed || (Date.now() - +dismissed) > 86400e3;
if (isWeChat && expired) showWeChatGuide();
  • · 仅当 UA 含 MicroMessenger 触发
  • · 全屏遮罩 rgba(0,0,0,.78) + backdrop-filter blur
  • · dismiss 后写 wechat_dismissed = Date.now() 到 localStorage
  • · 24 小时后再次访问会再提示一次
  • · 用 DOM API 创建,避开 innerHTML
为 · 什 · 么 · 这 · 么 · 做
  • PWA 安装受限 · 微信内 X5/WKWebView 不支持 beforeinstallprompt / 主屏添加
  • Service Worker 缺失 · 离线壳无法生效
  • Cookie / Storage 隔离 · 微信内 cookie 跨页面有时不持久,影响登录态
  • UI 适配差 · 微信底部导航条挤压视口,部分弹层被遮挡
业 · 界 · 参 · 考

掘金、语雀、Notion、知乎专栏、Twitter / X 都用同样的引导模式 —— 浮层提示 + 大箭头指右上角,文案聚焦"为什么要去外面浏览器"。 账房不做白屏弹窗(文案"为获得完整体验请使用浏览器打开"过于工程化), 而是用「📦 在浏览器中打开」搭配自定义"账房"棕青配色,贴近正文气质。
