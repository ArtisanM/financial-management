微 · 信 · 操 · 作 · 菜 · 单

点 ⋯ 后微信弹出操作菜单。「在浏览器打开」
在第二行第二个位置(图中金棕色高亮 + 缩放呼吸)。
点击后系统会启动默认浏览器加载同一个 URL。

为 · 什 · 么 · 要 · 跳 · 浏 · 览 · 器
微 · 信 · 内 · 限 · 制
  • · PWA 安装受限 — 不支持 beforeinstallprompt / 主屏添加
  • · Service Worker 缺失 — 离线壳无法生效
  • · Cookie / Storage 隔离 — 微信内 cookie 跨页有时不持久,登录态易失
  • · UI 适配差 — 底部导航条挤压视口,弹层位置错位
  • · 长按下载受限 — CSV 导出在微信内会跳转预览页而非下载
浏 · 览 · 器 · 完 · 整 · 体 · 验
  • · 可添加桌面图标(FR-34)→ 之后从主屏 1 秒打开
  • · Standalone 全屏体验,无微信 / 浏览器 chrome
  • · 登录态 7 天稳定持久
  • · CSV 导出能直接下载到本地"文件 App"
  • · 屏幕利用率最大(无微信顶栏占 88px)
触 · 发 · 检 · 测
// templates/fragments/layout.html (head 末尾)
const isWeChat = /MicroMessenger/i
                  .test(navigator.userAgent);
const dis = +localStorage
              .getItem('wx_dismissed') || 0;
const expired = Date.now() - dis > 86400 * 1000;

if (isWeChat && expired) {
  document.addEventListener('DOMContentLoaded',
    showWeChatGuide);
}
  • · 仅 UA 含 MicroMessenger 时触发
  • · 全屏遮罩 rgba(20,16,12,.78) + blur(2px)
  • · dismiss 后 24h 内不再骚扰,后再访问会再提示一次
  • · 用 DOM API 创建,避开 innerHTML
业 · 界 · 参 · 考

掘金 / 语雀 / Notion / 知乎专栏 / Twitter 都用同样的引导模式 —— 浮层提示 + 大箭头指右上角,文案聚焦"为什么去外面浏览器"。

账房不做白屏弹窗(文案"为获得完整体验请使用浏览器打开"过于工程化), 而是用「📦 在浏览器中打开」搭配自定义"账房"棕青配色,贴近正文气质。 家庭成员看到的是引导而非告警。
