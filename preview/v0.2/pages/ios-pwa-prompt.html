iOS Safari · 添加到主屏

iOS Safari 不支持 Chrome 的 beforeinstallprompt 自动弹窗,只能引导用户走系统分享菜单。 家庭账房在 iOS Safari 首次访问时弹出一个底部卡片,引导添加桌面图标 —— 添加后从主屏点击进入是 standalone 模式(无浏览器 UI),用起来像 App。

const isIOS = /iPad|iPhone|iPod/.test(navigator.userAgent);
const isSafari = /^((?!chrome|android|crios|fxios).)*safari/i
                   .test(navigator.userAgent);
const inStandalone = window.navigator.standalone === true;
const isWeChat = /MicroMessenger/i.test(navigator.userAgent);
const dismissed = +localStorage.getItem('pwa_dismissed') || 0;
const expired = Date.now() - dismissed > 7 * 86400e3;

if (isIOS && isSafari && !inStandalone && !isWeChat && expired) {
  setTimeout(showAddToHomeBanner, 1500);
}

业 · 界 · 参 · 考

Trello、Pinterest、Twitter Lite 都用底部固定卡片这一形态。 做底部 banner 而非中央 modal 的理由:中央 modal 阻塞内容, iOS 用户看到中央弹窗会下意识找 ✕ 关掉,而底部 banner 不阻塞,愿意看的就看。
